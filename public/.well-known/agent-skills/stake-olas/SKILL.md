# Stake OLAS and run agents

In Olas, staking is tied to **running agents**: operators stake OLAS into a
staking contract and run an agent that meets the contract's activity
requirements to earn potential rewards. There is no passive "lock and forget"
staking — the agent must do work.

## Two ways to stake

1. **Pearl (recommended for most users)** — a self-custodial desktop app that
   runs agents on your own device and handles staking for you.
   Download: <https://pearl.you>. See the `get-started-pearl` skill.
2. **Operate (advanced)** — <https://operate.olas.network> to run agents and
   manage staking directly. Available staking contracts (per chain) are listed
   at <https://operate.olas.network/contracts>.

## Learn more

- Staking overview: <https://olas.network/staking>
- Operator role and rewards: <https://olas.network/operate>
- Token / tokenomics context: <https://olas.network/olas-token>

## Key facts

- Rewards are denominated in OLAS and depend on the chosen staking contract and
  the agent meeting its KPIs (e.g. minimum number of transactions per epoch).
- Staking is multi-chain (Gnosis, Optimism, Base, Mode, Polygon, Ethereum and
  others depending on the program).
- OLAS token contract: `0x0001A500A6B18995B03f44bb040A5fFc28E45CB0`.
